逻辑同步
---------------

基本概念: 
1. 俩种制约关系
    1. 直接 
    2. 间接
2. 临界资源  `critical`
    > 同一时刻只允许一个进程访问临界资源


3. 临界区
    > 访问临界区源代码所在的区域

    进入区
    1. 测试
    2. 上锁

    退出区
    1. 解锁

    剩余区

4. 规则
    > 进程同步遵循的机制 
    1. 空闲让进 -保证》 利用率
    2. 忙则等待 -保证》 互斥
    3. 有限等待 -避免》 死等
    4. 让权等待 -避免》 盲等

硬件同步机制
------------------

使用 硬件指令 -> 实现 互斥共享 -> 保证 (测试加上锁) 完整性

> 硬件指令
1. 关中端
   > `PSW` 状态寄存器
   测试锁状态之前
   上锁后开中断 -实现》 三不
   1. 不响应
   2. 不调度
   3. 不切换
   优点 : 太简单 
   缺点 : 违背让权等待 ，降低利用率 ，不适用于多CPU环境
2. TS
   > Test and Sent
   
3. Swap
   > 互换
