> 实现进程同步,协调工作

1. 整型
    1. 整型变量 表示 可用资源
    2. 俩个原子的操作 wait signal
    优点 : 简单
    缺点 : 违背让权等待 
2. 记录型
    1. 请求资源失败时 - 阻塞 - 队列(唤醒)
    2. value 可用资源数量
    3. 队首指针表示阻塞队列
    缺点：
    可能导致 deadlock
3. and型
    1. 要么全分配，要么不分配
    2. wait -》 Swait  signal -》 Ssignal
4. 信号量型
    1. 低效 ，不安全
    2. 无法解决分配下限问题
